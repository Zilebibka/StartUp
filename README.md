## 1. Идея и проблемный контекст

- **Идея продукта:** Лёгкий музыкальный плеер в формате Mini App внутри Telegram для прослушивания, сортировки и организации личной музыки.
- **Проблема:** Пользователи хранят музыку в разрозненных источниках (память телефона, ВК, боты). Существующие Telegram‑боты позволяют только искать треки по запросу, но не дают возможности создавать подборки, сортировать треки или управлять плейлистами в привычном интерфейсе плеера.

**Сценарий:**
- **Пользователь/роль:** Молодой человек (18–30 лет), активный пользователь Telegram.
- **Ситуация:** Пользователь находит трек через бота (@vkm_bot) и скачивает его, либо пересылает себе. Файлы теряются в истории чатов. Чтобы послушать конкретный плейлист, приходится заново искать треки в боте или открывать стороннее приложение.
- **Проблема:** Отсутствие единого центра управления музыкой внутри Telegram. Нельзя сгруппировать найденные треки по настроению или жанру.
- **Последствия:** Трата времени на повторный поиск треков (время), неудобство прослушивания (качество опыта), отказ от прослушивания музыки в Telegram в пользу других приложений.

## 3. Выбор приоритетного сегмента

Сравнение сегмента **«Ботоводы»** по критериям:

| Критерий | Оценка |
|----------|--------|
| Выраженность и частота проблемы | Высокая (проблема «мусорки в чатах» проявляется каждый день при поиске трека) |
| Доступность сегмента | Высокая (мы знаем, где они сидят — в ботах-конкурентах) |
| Скорость принятия решения | Высокая (увидел удобный плеер — попробовал сразу) |
| Барьеры внедрения | Низкие (не нужно ничего скачивать, открывается в Telegram) |
| Вероятность оплаты | Средняя (платёжеспособность есть, но нужен сильный стимул) |

**Вывод:** выбран сегмент **«Ботоводы»** как стартовый, потому что у них есть осознанная боль, они уже в Telegram, и низкий порог входа.

## 6. Драйверы и барьеры

**Драйверы (почему спрос/актуальность растут):**
1. Рост аудитории Telegram — больше потенциальных пользователей.
2. Тренд на Mini Apps — Telegram развивает платформу, упрощая вход.
3. Усталость от подписок — пользователи ищут бесплатные или условно-бесплатные способы слушать музыку.
4. Увеличение объёма медиафайлов — растёт потребность в систематизации.

**Барьеры (что мешает входу/масштабированию):**
1. Привычка пользователей к текущему хаосу или стримингам.
2. Политика Telegram и авторские права — риск блокировок.
3. Технические ограничения Telegram API — возможно, не все функции плеера реализуемы.

## 7. Ценностное предложение

> **Для «Ботоводов»** (активных пользователей Telegram-ботов), которые в ситуации **«поиска трека, который они уже находили неделю назад»**, мы обеспечиваем **«единое хранилище и удобный плеер»**, улучшая **«скорость доступа к любимой музыке в 3 раза»** по сравнению с **«поиском в истории чатов»**.

**Основания (почему это реалистично):**
- Интеграция с Telegram — не требует выхода из приложения.
- Автоматическая сортировка треков по исполнителям (парсинг метаданных).
- Возможность импорта истории из ботов (упрощает переход).

## 9. Гипотезы бизнес-модели (10+)

1. В сегменте «Ботоводы» проблема потери треков возникает не реже 3 раз в неделю.
2. 70% пользователей Telegram-ботов готовы перейти в Mini App ради функции «плейлисты».
3. Пользователи старше 30 лет реже используют ботов для музыки (не наш сегмент).
4. Автоматическая сортировка треков по исполнителям важнее, чем создание собственных плейлистов.
5. Переход на наше решение возможен при условии, что пользователь может импортировать свою историю из бота.
6. Наиболее доступный канал контакта — таргетированная реклама на каналах с обложками альбомов (Music release).
7. При цене подписки 150 руб/мес хотя бы 5% бесплатных пользователей купят Premium.
8. Пользователи **не** готовы платить за разблокировку функции «Эквалайзер» (воспринимается как базовая).
9. Главное возражение: «А зачем, если есть Spotify?» — снимается демонстрацией проблемы: «Найди трек, который ты скинул вчера другу за 10 секунд».
10. Критическая функция для мобильной версии — Drag-and-drop для сортировки треков в плейлисте.
11. Без интеграции с облачными хранилищами (чтобы подтягивать файлы) продукт будет бесполезен.
